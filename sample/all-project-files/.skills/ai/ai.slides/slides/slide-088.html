개인정보 마스킹 자동화

1/2

학습 목표

주민번호·연락처·이메일·이름이 섞인 명단을 학교 표준 형식으로 자동 마스킹한다. AI에 입력 가능한 안전 형식으로 변환하는 게 목적이다.

학교 표준 마스킹 규칙

항목 원본 마스킹 후

이름 김민수 김○○ 또는 직책 A
사번 2024-12345 EMP-XX-XXX 또는 임시 코드
주민번호 880101-1234567 88****-*****
휴대폰 010-1234-5678 010-****-5678
이메일 abc@jnu.ac.kr a***@jnu.ac.kr
주소 광주광역시 북구 ○○동 ○○번지 광주광역시 북구 (이후 마스킹)

따라하기 — 마스킹 프롬프트

[역할] 너는 학교 인사 데이터의 개인정보 마스킹 자동화 도구다. [입력 데이터] (엑셀에서 복사한 명단 — 원본 그대로) 김민수 | 2024-12345 | 880101-1234567 | 010-1234-5678 | abc@jnu.ac.kr 이영희 | 2023-11111 | 920215-2345678 | 010-9876-5432 | def@jnu.ac.kr … [작업] 모든 행을 학교 표준 마스킹 규칙에 맞게 변환: - 이름 → 「성+○○」 형식 (예: 김○○) - 사번 → 「EMP-N1」 임시 코드 (행 순서대로 N1, N2, ...) - 주민번호 → 「YY****-*****」 - 휴대폰 → 「010-****-뒷4자리」 - 이메일 → 첫 글자만 노출 후 별표 (a***@도메인) [출력 형식] - 마스킹된 데이터를 TSV로 출력 - 마지막 행에 「변환: N건 / 마스킹 누락: 0건」 표시 [원칙] - 한 행이라도 마스킹 누락이 의심되면 「[검토 필요]」 표시 - 외부 도메인 이메일(@gmail.com 등)은 도메인까지 마스킹
